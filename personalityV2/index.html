How It Works

The Orchestration Loop

Every discussion follows a strict phase progression enforced by the Moderator. No phase is skipped. No shortcuts.

🔍
Clarify
Gather specs, identify open decisions
→
🧭
Explore
Propose approaches, generate options
→
🎯
Narrow
Converge on the best approach
→
⚙️
Execute
Build the concrete deliverable
→
📋
Debrief
Review outcomes, capture lessons
→
🏁
Stop
Deliverables complete, discussion ends
Key Features

Built for Real Work

Every feature exists to solve a real problem in multi-agent orchestration. Nothing is decorative.

🖥️
Local LLMs via Ollama
Runs entirely on local models through Ollama. No cloud dependencies, no API costs, full data privacy.
📊
PySide6 Real-Time GUI
Monitor agent states, iteration progress, and discussion flow in real time through a native desktop interface.
🔔
Discord Webhook Integration
Receive notifications for phase advances and key discussion milestones directly in your Discord channels.
✅
Verified Tool Execution
Agents use verified tools — write_file, read_file, list_files, web_search, web_fetch — with system‑confirmed execution.
🧑‍💼
Dynamic Expert Provisioning
The Moderator can summon temporary expert agents with specialized knowledge for niche problems, then dismiss them.
🔒
Strict Role Boundaries
Each agent has enforced rules — only the Moderator saves files, the Pragmatist drafts deliverables, the Librarian grounds facts. No role overlap.
