Local LLMs via Ollama

LLMPersonality runs entirely on local language models through Ollama. No cloud API calls, no per‑token costs, full data privacy. Every agent in the discussion — from the Moderator to the Provocateur — is powered by a local model you control.

🔒
Full Data Privacy
All discussion data stays on your machine. No prompts or outputs are sent to external services.
💰
Zero API Costs
No per‑token charges. Run as many discussions, iterations, and agents as you need without cost concerns.
🎛️
Model Flexibility
Swap models through Ollama's interface. Use different models for different agent roles if desired.

PySide6 Real-Time GUI

A native desktop interface built with PySide6 provides real‑time visibility into the discussion. Monitor agent states, track iteration progress, and observe phase transitions as they happen.

📊
Agent State Monitoring
See which agent is active, what phase the discussion is in, and how each agent contributes in real time.
🔄
Iteration Tracking
Follow the discussion across multiple rounds. Each iteration's context and progress is visible in the GUI.
⚡
Phase Transition Visibility
Watch the discussion move through Clarify → Explore → Narrow → Execute → Debrief → Stop in real time.

Discord Webhook Integration

Stay informed even when you’re not watching the GUI. Discord webhook integration sends notifications for phase advances and key discussion milestones directly to your channels.

🔔
Phase Advance Alerts
Get notified when the discussion moves from one phase to the next — e.g., Explore → Narrow → Execute.
📡
Milestone Notifications
Receive updates on key events: deliverable completion, expert summoning, discussion conclusion.
Verified Tools

System‑Confirmed Execution

Agents don’t just claim they did something — the system verifies it. Tool executions are confirmed with [SYSTEM VERIFIED] tags in the transcript.

write_file
Store generated text as a persistent file for later retrieval. Ensures the conversation folder exists and writes content. Overwrites if the file already exists. Only the Moderator can use this tool.
read_file
Load previously saved content so agents can use it in context. Retrieves stored conversation‑specific content like saved notes, drafts, or prior outputs.
list_files
Discover what persistent files already exist for the current conversation. Helps agents decide what content can be reused or loaded next.
web_search
Perform internet searches to find information. Primarily used by the Librarian to ground discussions in verified, external evidence.
web_fetch
Fetch the content of a specific web page by URL. Used by the Librarian to retrieve and verify source material.
summon_expert
Temporarily bring in a specialized agent with niche expertise for a specific challenge. Only the Moderator can use this tool.

Dynamic Expert Provisioning

Not every discussion needs the same expertise. When the core six agents encounter a problem outside their domains, the Moderator can summon a temporary expert agent — a specialist with targeted knowledge for the specific challenge.

🧑‍💼
Summoned by the Moderator
Only the Moderator can invoke summon_expert. The expert receives a role name and expertise description, then joins the next round.
⏱️
Temporary & Targeted
Experts are not permanent members. They join for the specific round where their expertise is needed, then are dismissed.
🎯
Context‑Specific
Each expert is defined with a precise role and expertise description tailored to the current discussion's needs.
